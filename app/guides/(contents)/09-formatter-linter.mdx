Форматер и линтер очень важная часть разработки, сегодня это необходимость для любого проекта.
Форматер отвечает за внешний вид кода, а линтер - за правила и потенциальные ошибки.

```js
import { eslint } from '@siberiacancode/eslint';

export default eslint({
  typescript: true,
  react: true
});
```

Минимальный набор: `prettier` для форматирования и `eslint` для проверок.
В команде это снижает количество споров о стиле, ускоряет ревью и ловит ошибки раньше.
Чтобы эффект был стабильным, подключайте проверки в CI и pre-commit: тогда правила применяются одинаково у всех.
