Единый формат коммитов нужен не для "красоты", а для скорости команды.
С ним проще читать историю, быстрее делать ревью и автоматизировать changelog.
Особенно это важно в больших репозиториях, где по истории коммитов часто ищут причины изменений и точки регрессий. Можно взять готовый стандарт (Conventional Commits) или договориться о своем коротком формате.
Главное - использовать один и тот же шаблон во всех PR.

## Кастомный шаблон:

```bash
{номер таски} {персональный emoji} {сообщение}

#1 🧊 init commit
```

## Conventional Commits:

```bash
feat(auth): init commit
```

Зафиксируйте правила в репозитории и применяйте их последовательно.
Даже простой, но стабильный формат лучше, чем случайные сообщения.
Отдельно полезно договориться о глаголах (`feat`, `fix`, `refactor`, `docs`) и масштабе одного коммита - это резко улучшает навигацию по истории.
